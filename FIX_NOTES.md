# 自动补牌流程修复说明

## 修改文件
- `src/App.tsx`

## 修复目标
修复完整交锋结束后出现“日志显示 REPLENISH COMPLETE，但玩家与 AI 实际没有补牌”的问题。

## 主要调整
1. 重构结算后的状态流：
   `resolving -> move-to-discard -> replenishing -> replenish-complete -> round-end -> next-turn`
2. 使用最新状态快照与统一计时器队列，避免 React 异步状态和旧闭包造成补牌计算错误。
3. 公共牌库充足时：玩家与 AI 按各自缺牌数量补牌，单方最多补 2 张，手牌上限 4 张。
4. 公共牌库不足时：保留“下一阶段主场方先抽，双方逐张交替抽取”的残局规则。
5. 公共牌库为 0 时：跳过补牌，进入最终交锋状态。
6. 重开游戏或返回大厅时：清理结算计时器、动画与提示，避免旧回合逻辑污染新对局。

## 建议优先手动验收
1. 玩家开局打出 3 张同类牌，AI 空过：玩家应从 1 张补到 3 张，公共牌库减少 2。
2. 玩家打出 2 张，AI 打出 2 张：双方各补 2 张，公共牌库依次减少 4。
3. 公共牌库仅剩 3 张且双方均缺 2 张：按下一阶段主场方优先、双方交替抽牌。
